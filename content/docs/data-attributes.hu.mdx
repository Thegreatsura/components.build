---

Adat-attribútumok erőteljes módot biztosítanak a komponens állapotának és felépítésének kitettségére a fogyasztók számára, lehetővé téve a rugalmas stílusozást prop-explózió nélkül. A modern komponenskönyvtárak két fő mintát használnak: `data-state` a vizuális állapotokhoz és `data-slot` a komponensazonosításhoz.

## Állapot stílusozása `data-state`-tel

Az egyik leggyakoribb rossz gyakorlat a komponensek stílusozásában, hogy különböző állapotokhoz külön className propokat tesznek elérhetővé.

Kevésbé modern komponenseknél gyakran ilyen API-kat láthatsz:

```tsx
<Dialog
  openClassName="bg-black"
  closedClassName="bg-white"
  classes={{
    open: "opacity-100",
    closed: "opacity-0"
  }}
/>
```

Ennek a megközelítésnek több problémája van:
- Összekapcsolja a komponens belső állapotát a stílusozási API-val
- A propok számának robbanásszerű növekedését eredményezi, ahogy a komponensek bonyolultabbá válnak
- Megnehezíti a komponens használatát és karbantartását
- Megakadályozza az állapotok kombinációi alapján történő stílusozást

### A megoldás: `data-state` attribútumok

Ehelyett használj `data-*` attribútumokat a komponens állapotának deklaratív kitettségére. Ez lehetővé teszi a fogyasztók számára, hogy külső CSS szelektorokkal stílusozzák a komponenseket az állapot alapján:

```tsx title="component.tsx"
const Dialog = ({ className, ...props }: DialogProps) => {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <div
      data-state={isOpen ? 'open' : 'closed'}
      className={cn('transition-all', className)}
      {...props}
    />
  );
};
```

Most a fogyasztók kívülről tudják stílusozni a komponenst az állapot alapján:

```tsx title="app.tsx"
<Dialog className="data-[state=open]:opacity-100 data-[state=closed]:opacity-0" />
```

### Ennek a megközelítésnek az előnyei

1. **Egyetlen className prop** - Nincs szükség több, állapotspecifikus className progra
2. **Komponálható** - Kombinálhatsz több adat-attribútumot összetett állapotokhoz
3. **Standard CSS** - Működik bármilyen CSS-in-JS megoldással vagy sima CSS-sel
4. **Típusbiztos** - A TypeScript képes kikövetkeztetni az adat-attribútumok értékeit
5. **Ellenőrizhető** - Az állapotok láthatóak a DevTools-ban HTML attribútumokként

### Gyakori állapotminták

Használj adat-attribútumokat mindenféle komponensállapotnál:

```tsx
// Open/closed state
<Accordion data-state={isOpen ? 'open' : 'closed'} />

// Selected state
<Tab data-state={isSelected ? 'active' : 'inactive'} />

// Disabled state (in addition to disabled attribute)
<Button data-disabled={isDisabled} disabled={isDisabled} />

// Loading state
<Button data-loading={isLoading} />

// Orientation
<Slider data-orientation="horizontal" />

// Side/position
<Tooltip data-side="top" />
```

### Stílusozás Tailwind-del

A Tailwind támogatja a tetszőleges variánsokat, így az adat-attribútumokkal való stílusozás elegáns:

```tsx
<Dialog
  className={cn(
    // Base styles
    'rounded-lg border p-4',
    // State-based styles
    'data-[state=open]:animate-in data-[state=open]:fade-in',
    'data-[state=closed]:animate-out data-[state=closed]:fade-out',
    // Multiple attributes
    'data-[state=open][data-side=top]:slide-in-from-top-2'
  )}
/>
```

Gyakran használt állapotokhoz kiterjesztheted a Tailwind konfigurációját:

```js title="tailwind.config.js"
module.exports = {
  theme: {
    extend: {
      data: {
        open: 'state="open"',
        closed: 'state="closed"',
        active: 'state="active"',
      }
    }
  }
}
```

Most már használhatsz rövidített szintaxist:

```tsx
<Dialog className="data-open:opacity-100 data-closed:opacity-0" />
```

### Integráció a Radix UI-val

Ezt a mintát széles körben használja a [Radix UI](https://www.radix-ui.com/), amely automatikusan alkalmazza az adat-attribútumokat az elemeire:

```tsx
import * as Dialog from '@radix-ui/react-dialog';

<Dialog.Root>
  <Dialog.Trigger />
  <Dialog.Portal>
    {/* Radix automatically adds data-state="open" | "closed" */}
    <Dialog.Overlay className="data-[state=open]:animate-in data-[state=closed]:animate-out" />
    <Dialog.Content className="data-[state=open]:fade-in data-[state=closed]:fade-out" />
  </Dialog.Portal>
</Dialog.Root>
```

A Radix által biztosított egyéb adat-attribútumok közé tartozik:
- `data-state` - open/closed, active/inactive, on/off
- `data-side` - top/right/bottom/left (pozícionált elemekhez)
- `data-align` - start/center/end (pozícionált elemekhez)
- `data-orientation` - horizontal/vertical
- `data-disabled` - jelen van, ha le van tiltva
- `data-placeholder` - jelen van, ha helykitöltő látható

## Komponensazonosítás `data-slot`-tal

Míg a `data-state` a vizuális állapotokat követi, a `data-slot` a komponens típusát azonosítja egy összetételen belül. Ezt a mintát népszerűsítette a [shadcn/ui](https://ui.shadcn.com/), és lehetővé teszi a szülő komponensek számára, hogy célzottan stílusozzanak bizonyos gyermekkomponenseket anélkül, hogy törékeny class nevekre vagy elemcímkékre támaszkodnának.

### A gyermekek célzásának problémája

A gyerekkomponensek stílusozásának hagyományos megközelítései jelentős korlátokkal rendelkeznek:

```tsx
// Relies on element types - breaks if implementation changes
<form className="[&_input]:rounded-lg [&_button]:mt-4" />

// Relies on class names - breaks if classes change
<form className="[&_.text-input]:rounded-lg" />

// Requires passing classes through props - verbose
<form>
  <input className={inputClasses} />
  <button className={buttonClasses} />
</form>
```

### A megoldás: `data-slot` attribútumok

Használd a `data-slot`-ot, hogy a komponensek stabil azonosítókat kapjanak, amelyeket a szülők célozhatnak:

```tsx title="field-set.tsx"
function FieldSet({ className, ...props }: React.ComponentProps<"fieldset">) {
  return (
    <fieldset
      data-slot="field-set"
      className={cn(
        "flex flex-col gap-6",
        // Target specific child slots
        "has-[>[data-slot=checkbox-group]]:gap-3",
        "has-[>[data-slot=radio-group]]:gap-3",
        className
      )}
      {...props}
    />
  );
}
```

```tsx title="checkbox-group.tsx"
function CheckboxGroup({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="checkbox-group"
      className={cn("flex flex-col gap-2", className)}
      {...props}
    />
  );
}
```

### A `data-slot` előnyei

1. **Stabil azonosítók** - Nem törnek el, amikor a megvalósítás részletei megváltoznak
2. **Szemantikus célzás** - A komponens célja alapján céloz, nem a szerkezete alapján
3. **Elhatárolás** - A belső classok privátok maradnak
4. **Komponálhatóság** - Tetszőleges beágyazással és kompozícióval működik
5. **Típusbiztos** - Ellenőrizhető és dokumentálható

### `has-[]` használata szülő-érzékeny stílusozáshoz

A Tailwind `has-[]` szelektora a `data-slot`-tal kombinálva erőteljes szülő-érzékeny stílusozást tesz lehetővé:

```tsx title="form.tsx"
function Form({ className, ...props }: React.ComponentProps<"form">) {
  return (
    <form
      data-slot="form"
      className={cn(
        "space-y-4",
        // Adjust spacing when specific slots are present
        "has-[>[data-slot=form-section]]:space-y-6",
        "has-[>[data-slot=inline-fields]]:space-y-2",
        // Style based on slot states
        "has-[[data-slot=submit-button][data-loading=true]]:opacity-50",
        className
      )}
      {...props}
    />
  );
}
```

### `[&_]` használata leszármazott célzáshoz

Mélyebb beágyazáshoz használd a `[&_selector]` mintát, hogy bármely leszármazottat célba vegyél:

```tsx title="card.tsx"
function Card({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="card"
      className={cn(
        "rounded-lg border p-4",
        // Target any descendant with data-slot
        "[&_[data-slot=card-header]]:mb-4",
        "[&_[data-slot=card-title]]:text-lg [&_[data-slot=card-title]]:font-semibold",
        "[&_[data-slot=card-description]]:text-sm [&_[data-slot=card-description]]:text-muted-foreground",
        "[&_[data-slot=card-footer]]:mt-4 [&_[data-slot=card-footer]]:border-t [&_[data-slot=card-footer]]:pt-4",
        className
      )}
      {...props}
    />
  );
}
```

### Globális CSS `data-slot`-tal

A data slotok szépen működnek globális CSS-sel a témaszintű konzisztencia érdekében:

```css title="globals.css"
/* Style all buttons within forms */
[data-slot="form"] [data-slot="button"] {
  @apply w-full sm:w-auto;
}

/* Style submit buttons specifically */
[data-slot="form"] [data-slot="submit-button"] {
  @apply bg-primary text-primary-foreground;
}

/* Adjust inputs within inline layouts */
[data-slot="inline-fields"] [data-slot="input"] {
  @apply flex-1;
}

/* Style based on state combinations */
[data-slot="dialog"][data-state="open"] [data-slot="dialog-content"] {
  @apply animate-in fade-in;
}
```

### Elnevezési konvenciók

Kövesd ezeket a konvenciókat a konzisztens `data-slot` elnevezéshez:

1. **Használj kebab-case-t** - `data-slot="form-field"` ne `data-slot="formField"`
2. **Légy specifikus** - `data-slot="submit-button"` ne `data-slot="button"`
3. **Illeszkedjen a komponens céljához** - A név tükrözze, mit csinál, ne azt, hogyan néz ki
4. **Kerüld a megvalósítási részleteket** - `data-slot="user-avatar"` ne `data-slot="rounded-image"`

```tsx
// Good examples
data-slot="search-input"
data-slot="navigation-menu"
data-slot="error-message"
data-slot="submit-button"
data-slot="card-header"

// Avoid
data-slot="input"           // Too generic
data-slot="blueButton"      // Includes styling
data-slot="div-wrapper"     // Implementation detail
data-slot="mainContent"     // Use camelCase
```

## Mikor használj adat-attribútumokat és mikor propokat

Azt, hogy mikor melyik mintát használd, kulcsfontosságú a tiszta API-hoz:

### `data-state` használati esetek
- **Vizuális állapotok** - open/closed, active/inactive, loading stb.
- **Elrendezési állapotok** - orientation, side, alignment
- **Interakciós állapotok** - hover, focus, disabled (amikor a gyermekek stílusozására van szükség)

### `data-slot` használati esetek
- **Komponensazonosítás** - Stabil azonosítók célzásra
- **Kompozíciós minták** - Szülő-gyerek viszonyok
- **Globális stílusozás** - Téma-szintű komponensstílusok
- **Variánstól független célzás** - Bármely komponens variánsának célzása

### `props` használati esetek
- **Variánsok** - Különböző vizuális tervek (primary, secondary, destructive)
- **Méret** - sm, md, lg
- **Viselkedési konfiguráció** - controlled/uncontrolled, alapértelmezett értékek
- **Eseménykezelők** - onClick, onChange stb.

### Kombinált megközelítés

Egy jól megtervezett komponens mindhárom mintát megfelelően használja:

```tsx title="button.tsx"
type ButtonProps = {
  variant?: 'primary' | 'secondary' | 'destructive';
  size?: 'sm' | 'md' | 'lg';
  loading?: boolean;
  disabled?: boolean;
  onClick?: () => void;
  className?: string;
};

const Button = ({
  variant = 'primary',
  size = 'md',
  loading,
  disabled,
  className,
  ...props
}: ButtonProps) => {
  return (
    <button
      // Slot for targeting
      data-slot="button"
      // State for conditional styling
      data-loading={loading}
      data-disabled={disabled}
      className={cn(
        // Variant styles via props
        buttonVariants({ variant, size }),
        // Additional state styling allowed via className
        className
      )}
      disabled={disabled}
      {...props}
    />
  );
};
```

Mostantól a gomb többféleképpen használható és stílusozható:

```tsx
// Basic usage with variants
<Button variant="primary" size="lg">Submit</Button>

// Parent targeting via data-slot
<form className="[&_[data-slot=button]]:w-full">
  <Button>Submit</Button>
</form>

// State-based styling via data-state
<Button
  loading={isLoading}
  className="data-[loading=true]:opacity-50"
>
  Submit
</Button>

// Global CSS can target any button
// [data-slot="button"][data-loading="true"] { ... }
```

Az adat-attribútumok robusztus alapot biztosítanak a modern komponenskönyvtárak stílusozásához. A `data-state` használatával a vizuális állapotokhoz és a `data-slot` használatával a komponensazonosításhoz rugalmas, karbantartható API-t hozol létre, amely skálázódik az egyszerű komponensektől a komplex designrendszerekig.
